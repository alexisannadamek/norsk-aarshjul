# Norsk Årshjul

## Event Style Guide v1.1

**Status:** Approved

---

# Philosophy

Every event should answer three questions:

1. **What is today?**
2. **Why does it matter?**
3. **How might someone participate?**

The calendar should teach naturally through everyday use rather than feeling like homework.

---

# Event Title

**Format**

```text
<Category Emoji> English Name
Norwegian Name
```

Example:

```text
🏛️ Constitution Day
Grunnlovsdagen
```

### Rules

- One category emoji only.
- English name first.
- Norwegian name on its own line.
- No abbreviations.
- No parentheses.

---

# Event Description

**Target length:** 80–120 words

### Writing style

- Warm
- Informative
- Neutral
- Historically accurate
- Easy to read

Avoid opinions and unnecessary adjectives.

---

# Section Order

Every event follows this structure whenever applicable.

## Description

One short paragraph introducing the event.

Example:

> Norway celebrates the signing of its Constitution on 17 May 1814. It is the country's National Day and is marked with children's parades, bunads, music, and community celebrations.

---

## 🎉 Traditions

Maximum **five** bullet points.

Only include traditions genuinely associated with the event.

Example:

```text
🎉 Traditions

• Fly the Norwegian flag
• Watch parades
• Eat ice cream
• Wear a bunad
• Spend time with family
```

---

## 📖 Norwegian

Exactly **one** useful vocabulary word or phrase.

Example:

```text
📖 Norwegian

Grunnlov = Constitution
```

No pronunciation guides.

No grammar lessons.

---

## ℹ️ Notes

Include only when necessary.

Examples:

```text
🚩 Official Norwegian Flag Day
```

```text
Modern reconstructed observance.
```

```text
Date varies with Easter.
```

If there are no notes, omit this section entirely.

# Project Categories

The project contains exactly six categories.

## 🏛️ Norwegian Civic

Modern Norway.

Includes:

- Public holidays
- National observances
- Government commemorations

---

## 👑 Royal Family

Includes:

- Birthdays
- Jubilees
- Coronations
- Royal commemorations

---

## 🌲 Traditional Norway

Includes:

- Primstav
- Historic folk customs
- Agricultural traditions
- Traditional feast days

---

## ⚔️ Norse / Ásatrú

Modern reconstructed Norse observances and historically documented Norse festivals.

Every reconstructed observance will be clearly identified.

---

## 🌞 Astronomy

Includes:

- Solstices
- Equinoxes
- Other astronomical events directly relevant to the Norwegian year

---

## 🌿 Seasonal Norway

Nature-focused observances.

Includes:

- Berry season
- Mushroom season
- Harvest
- Aurora season
- Traditional foods
- Seasonal reminders

---

# Emoji Rules

Category emojis are used for organization.

Approved category emojis:

- 🏛️ Norwegian Civic
- 👑 Royal Family
- 🌲 Traditional Norway
- ⚔️ Norse / Ásatrú
- 🌞 Astronomy
- 🌿 Seasonal Norway

Additional emojis may be used sparingly inside event descriptions.

Avoid using country flag emojis or regional indicator emojis as category markers to ensure consistent rendering across Windows, macOS, Linux, GitHub, iOS, and Android.

---

# Dates

Priority order:

1. Official recurrence rules
2. Astronomical calculation
3. Historical fixed date
4. Modern consensus

Never invent recurrence rules simply because they are easier.

---

# Internal Evidence Levels

These values are internal metadata and are not shown to users.

| Level | Meaning |
|-------|---------|
| Official | Government or official institution |
| Historical | Supported by historical sources |
| Traditional | Longstanding Norwegian folk tradition |
| Reconstruction | Modern reconstruction based on historical evidence |
| Seasonal | Modern ecological or seasonal marker |

---

# Writing Standards

Always:

- Present tense
- Active voice
- Neutral wording
- Short paragraphs
- Consistent punctuation

Never:

- Use clickbait
- Use exclamation points in descriptions
- Tell jokes inside event descriptions
- Include political commentary
- Present speculation as fact

---

# Vocabulary Standards

Exactly one vocabulary item.

Example:

```text
📖 Norwegian

Snø = Snow
```

---

# Traditions Standards

Traditions should be:

- Historically documented whenever possible
- Widely practiced or culturally recognized
- Reasonable for someone outside Norway to participate in

---

# Sources

Sources do **not** appear inside calendar events.

Each event will instead contain an internal identifier linking to documentation in the repository.

The repository will include a dedicated `sources.md` documenting every source used.

---

# Event Length

Target:

**150–250 words maximum**, including all sections.

Most events should take under one minute to read.

---

# Version Metadata

Every event will include hidden internal metadata:

- Version introduced
- Category
- Evidence level
- Last reviewed
- Last updated

These fields live in the project database, not in the calendar itself.

# Official Project Standard

This document is the canonical formatting standard for every event in **Norsk Årshjul — Living the Norwegian Year**.

Every event—whether it is Constitution Day, Sankthans, Yule, the Spring Equinox, or blueberry season—will follow this structure unless a future version of the style guide is intentionally adopted.

### Version 1.1 Changes

- Removed country flag emojis from project standards to ensure consistent rendering across all operating systems.
- Standardized the six official project category emojis.
- Established the permanent writing, formatting, and metadata standards for all future events.
